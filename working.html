Working of HTML

Request and Response cycle
When you type a URL you make a request and in response, the browser shows the correct or appropriate file.
The IP address determined by the browser creates an HTTP request, there is a lot of information in this request.
The server returns the file as response according to the requests you made.
If the server can't complete your request it will show you an error.



URL part
  1. Protocol: Shows how to connect
    • Examples:
      • HTTP
      • Hypertext Transfer Protocol
      • HTTPS
      • Secure Hypertext Transfer Protocol
      • FTP
      • File Transfer Protocol
  1. Domain Names: Identifies the entity you want to connect
    • Each URL have different top-level Domain
    • Dertermined by ICAAN
      • The Internet Corporation for Assigned Names and Numbers
      • for info click here
    • DNS: Domain Name Server
      • DNS will look up the IP Address according to the URL we typed.
    • Examples:
      • xyz.edu
      • abc.com
      • mn.bizz
      • wikipidia.org
  1. Document: URLs can specify the Documents.
    • This is an optional idea.
    • We can open exactly the page we want using Document as address.
    • If we do not mention the document then default look of the site is opened.
  1. IP Adderess
    • It is a unique address to give to page or URL.
    • Every computer has its IP.
    • IPv6
      • Internet protocol version 6
      • It can look like this
        • XXXX:XXXX:XXXX:XXXX:XXXX:XXXX:XXXX:XXXX
        • X can have 16 Different values, this makes more than 3000 trillion unique combinations.
    • DNS will lookup for the unique IP based on your typed URL.
